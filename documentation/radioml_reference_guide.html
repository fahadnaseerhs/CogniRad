01 · What is RadioML 2018.01a?

A large, publicly available dataset of real radio signals captured using physical Software Defined Radio (SDR) hardware by DeepSig. Because it comes from real antennas — not math simulations — it includes real-world impairments like frequency offset, phase noise, and timing jitter. This makes models trained on it actually work in deployment.

It is the standard academic benchmark — cited in 300+ IEEE papers — so you can compare your accuracy numbers directly against published results.

02 · IQ Samples — the core data format

Every digitized radio signal is stored as IQ samples. A sine wave at any instant can be described by two things:

I — In-phase component

The real part of the signal. Think of it as the projection of the signal onto the horizontal axis of a circle.

Q — Quadrature component

The imaginary part, shifted exactly 90° from I. Projection onto the vertical axis.

Together, one (I, Q) pair is a complete snapshot of the signal at one instant. A sequence of 1024 such pairs captures the full waveform in time.

s(t) = I·cos(2πf·t) − Q·sin(2πf·t)

This is how I and Q reconstruct the original signal at carrier frequency f.

Worked Example — Reading an IQ sample

Say a BPSK transmitter sends a single bit. The dataset stores this as:

sample[0] = ( I=+0.707 , Q=+0.000 )
sample[1] = ( I=+0.693 , Q=+0.141 )
sample[2] = ( I=+0.650 , Q=+0.277 )
      ⋮
sample[1023] = ( I=+0.707 , Q=−0.001 )

Plotting these 1024 pairs on a 2D plane (I on x-axis, Q on y-axis) gives you the constellation diagram. For BPSK you would see two tight clusters — one at (+1, 0) and one at (−1, 0) — because BPSK only has two phase states.

Each dot above is one (I, Q) pair from the 1024-step sequence. The two clusters tell the CNN instantly: this is BPSK, not QAM, not FM.

03 · SNR — Signal-to-Noise Ratio

SNR measures how much useful signal power exists relative to background noise. Expressed in dB.

SNR (dB) = 10 · log₁₀ ( P_signal / P_noise )
+30 dB
very clean, easy to classify
+10 dB
mostly clean
0 dB
signal = noise (hard)
−10 dB
noise dominates
−20 dB
100× noise power — near-blind
Why include low-SNR data? Real deployments face bad channel conditions. Training on all 26 SNR levels (−20 dB to +30 dB) forces the model to learn robust features, not just cherry-pick clean signals.
04 · 24 Modulation Types

Modulation = how information is encoded onto a carrier wave. Each type has a distinct IQ signature, which is what the CNN learns to distinguish.

FamilyTypesWhat varies
Analog OOK, AM-SSB, AM-DSB, FM, WBFM amplitude or frequency continuously
Phase (PSK) BPSK, QPSK, 8PSK … 64PSK phase only (discrete steps)
Amp+Phase (QAM) 16QAM, 32QAM … 1024QAM both amplitude and phase
Frequency (FSK) 2FSK, 4FSK, 8FSK, 16FSK frequency hops between states
Spread Spectrum OFDM variants spread across many sub-carriers
05 · How CogniRad uses this dataset

CogniRad does not classify modulation types. Instead, the CNN uses the IQ patterns to classify channel state:

Free

Low energy, no meaningful signal. IQ values near zero.

Busy

Legitimate signal present. IQ matches known modulation patterns.

Jammed

Anomalous IQ — high power, broadband noise, no clear modulation structure.

Jamming signals have a characteristic IQ signature — the constellation plot smears across the plane with no clusters. The model learns to detect this pattern from the dataset's SNR and modulation variety.
